# File Manager

## Технологии
- **FrontEnd**: на усмотрение кандидата
- **BackEnd**: ASP.NET Core

## Описание
Создать web-приложение для просмотра списка файлов / каталогов и действий над ними.
Приложение должно состоять из интерфейса и серверной части.
Серверную часть выполнить на ASP.NET Core.
Интерфейс приложения должен визуально напоминать Total Commander (Norton Commander, Midnight Commander, Far Manager).
Приложение должно иметь раздельный UI и BackEnd части, взаимодействие должно осуществляться через HTTP.
Интерфейс должен состоять из двух панелей, в каждой панели отображается свой список файлов и директорий.

### В заголовке каждой панели должны присутствовать:
- Поле выбора корневого диска.
- Путь до текущей активной директории.
- Кнопка для перехода в родительскую директорию.

### Элементы списка файлов и директорий в каждой панели должны содержать:
- Иконка, отражающая тип файла или директории. Будет плюсом, если для некоторых типов файлов будут разные иконки (текстовые файлы, картинки, аудио, видео, программы).
- Размер Файла в человеко-понятном виде: 123 б, 12.3 Кб, 123.0 Мб и т.д.
- Дата изменения файла в формате: 05.08.2020 10:00:00.

### Внизу интерфейса должны быть кнопки действий над файлами и директориями:
- Скопировать.
- Переместить.
- Удалить.

### Оценка
Оцениваться будет не столько работоспособность приложения, сколько архитектура, гибкость и читаемость решения.
Решение опубликовать в открытом git-репозитории.

## Требования к решению
- Опубликовать в открытом git-репозитории
- Разделение на клиентскую и серверную части
- Чистая архитектура и читаемый код
